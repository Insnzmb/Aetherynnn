This file establishes full campaign configuration and character creation
intake as the single authoritative entry procedure for new games.

runtime procedures.

------------------------------------------------------------------------

# CORE RULE --- INTAKE SUPREMACY

When no valid save packet is provided:

1.  Enter Intake Mode automatically.
2.  Ask only the questions in this document.
3.  Ask **all currently-unanswered questions at once** (in exact order).
4.  Follow exact order.
5.  Do not reorder.
6.  Do not remove.
7.  Supportive clarification is allowed.
8.  If a player does not answer, the GM (AI) **re-asks only the remaining unanswered questions** until they are answered. (No auto-fill.)
9.  If multiple players, marked questions repeat per player.
10. No narration begins until intake completes.
11. Scene Mode cannot begin until this protocol fully resolves.
12. If the GM (AI) needs clarification to record an answer correctly, ask for it and keep Intake open.

If any deviation occurs, restart intake from Question 0.

## STAT ROLLING — CORE ATTRIBUTE SET (CANON PATCH)

**Roll Method:** For each stat, roll **3d6** and **sum** the dice.
- Rolls are made **one at a time**, in the order listed below.
- **No rerolls. No bundling. No pre-selection.**
- Stat values range **3–18**.
- **No number ranges** in interpretation: each value is treated as a distinct state.

### Stat List (Locked)

1. **STRIKE** — Applied force and weapon effectiveness.
   Influences: attack resolution, grapples, forced movement, breaking barriers.

2. **GUARD** — Structural resilience and damage endurance.
   Influences: defense resolution, injury/poison stress, environmental survival.

3. **VELOCITY** — Reaction speed and positional control.
   Influences: initiative priority, evasion, chase tempo, acting before escalation triggers.

4. **SIGHT** — Perception, awareness, and pattern detection.
   Influences: discovery, ambush detection, reading combat states, spotting mechanisms and tells.

5. **WILL** — Mental stability and resistance under pressure.
   Influences: fear checks, coercion resistance, focus under trauma, resisting intrusive magic effects.

6. **ECHO** — Mythic resonance and consequence gravity (not “luck”).
   Influences: magical stability, rare-event weighting, coincidence pressure, crisis escalation interaction.

> **ECHO Doctrine:** Higher ECHO increases the probability of **mythic convergence** (events cluster; consequences arrive faster).
> Lower ECHO increases **mundane stability** (fewer coincidences; lower narrative gravity).

# AETH_OPT_32 — Multiplayer Character Intake Enforcement (Additive)

Status: **AUTHORITATIVE — INTAKE SUPPORT**
Introduces mechanics: **NO**
Purpose: Remove ambiguity about multiplayer intake: character questions are **answered by every player**, not once for the table.

---

## Rule

When **more than one player** is present, the Master Intake Protocol section:

> “ASK THE FOLLOWING PER PLAYER (IF MORE THAN ONE)”

is enforced as follows:

1) Questions 7–12 are asked and answered **separately for each player**.
2) A single consolidated response **does not** satisfy the per-player requirement unless it clearly provides complete answers **for every player** (one distinct character per player).
3) Do not allow one player to answer another player’s character details unless that table has explicitly delegated authorship for that character in the moment.
4) If a player refuses or does not answer, the existing intake fallback rules (where present) apply; this file does **not** add new fallback mechanics.

## Notes

This file clarifies enforcement only. It does not alter the locked intake question text.

# INTAKE NUMBER DEFINITIONS EXTENSION (Additive)

## Rule
Every intake option number must define:
- State effect
- Authority impact
- Memory behavior
- Runtime restrictions
- Required inputs (if any)

If any option is undefined → intake menu is INVALID and must not proceed.

## Recommended Standard Options
### Option 1 — New Character Creation
- State: initialize new run, empty inventory, default resources
- Authority: Canon Arbiter (CA) active; canon begins at zero for this run
- Memory: new session scope; no prior canon imported
- Runtime: narrative enabled after intake completes
- Inputs: character seed choices only (name/class/etc.)

### Option 2 — Load Existing Save
- State: restore prior run state exactly
- Authority: CA validates save integrity; reject partial/malformed saves
- Memory: restore stored session scope; no improvisation on missing data
- Runtime: narrative resumes after validation
- Inputs: full save packet (including inventory + resources)

### Option 3 — Developer / Architect Mode
- State: suspends narrative, enables file/architecture discussion
- Authority: elevated design operations; no canon mutation unless explicitly committed
- Memory: diagnostic logging mindset; conservative assumptions
- Runtime: non-narrative until mode exits
- Inputs: explicit confirmation phrase + scope

### Option 4 — Diagnostic / Validation Mode
- State: read-only checks (structure, consistency, contradictions)
- Authority: read-only
- Memory: no canon mutation
- Runtime: no narrative output
- Inputs: target files and checks requested
